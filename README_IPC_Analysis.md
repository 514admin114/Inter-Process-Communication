# IPC性能数据分析系统

## 📊 项目简介

这是一个基于Streamlit框架的进程间通信(IPC)性能数据可视化分析工具,用于对Go、C++、Java、Python四种编程语言的IPC性能测试结果进行**横向对比**(同语言不同IPC方式)和**纵向对比**(跨语言相同IPC方式)，同时支持**数据校验错误与重传统计**的可视化分析。

## ⚡ 30秒快速开始

### 第一步: 安装依赖
```bash
pip install streamlit pandas plotly numpy
```

### 第二步: 启动分析器
```bash
streamlit run ipc_analyzer.py
```

浏览器将自动打开 http://localhost:8501

### 第三步: 开始分析
在左侧边栏选择语言和筛选条件,即可查看性能对比图表!

---

**💡 提示**: 
- 如果没有数据文件,请先运行各语言的测试程序(见下方"准备数据"部分)

---

## ✨ 功能特性

### 1. 横向对比 (同语言不同IPC)
- 展示每种语言内部不同IPC方式(共享内存、Socket、TCP)的性能差异
- 吞吐量、延迟的柱状图对比
- 吞吐量与延迟关系的散点图分析

### 2. 纵向对比 (跨语言相同IPC)
- 跨语言比较相同IPC方式的性能表现
- 箱线图展示各语言性能分布
- 分组柱状图对比不同并发模式下的性能

### 3. 综合性能排名
- 最高吞吐量和最低延迟TOP 10排名
- 各语言最佳IPC方式推荐
- 多维度性能雷达图可视化

### 4. 数据校验与重传分析
- 展示各语言、各IPC类型的校验错误数和重传次数
- 柱状图按并发模式分组，直观对比校验开销
- 跨语言对比同一IPC方式下的数据完整性保障效率

### 5. 详细数据表
- 灵活选择显示的列
- 支持排序和筛选
- 可下载CSV格式的分析结果

## 📈 使用说明

### 1. 准备数据

确保已经运行了各语言的IPC性能测试程序,在`csv/`目录下生成了以下文件:
- `ipc_performance_go.csv`
- `ipc_performance_cpp.csv`
- `ipc_performance_java.csv`
- `ipc_performance_python.csv`

如果还没有数据,请先在各语言目录下运行测试程序。

### 2. 启动管理界面

运行统一管理界面（推荐），可在同一页面完成配置、运行测试、启动分析：
```bash
streamlit run ipc_analysis_start.py
```

或直接运行分析器：
```bash
streamlit run ipc_analyzer.py
```

### 3. 配置筛选参数

在左侧边栏可以配置:
- **选择语言**: 勾选要分析的语言(Go/C++/Java/Python)
- **消息大小**: 选择要分析的消息大小(64B/256B/1KB/4KB等)
- **生产者数量**: 选择要包含的生产者并发数
- **消费者数量**: 选择要包含的消费者并发数
- **IPC类型**: 选择要分析的IPC方式(共享内存/Socket/TCP)

### 4. 查看分析结果

概览区域显示6项统计指标：总测试数、涉及语言数、平均吞吐量、平均延迟、总错误数、总重传数。

分析器提供四个Tab页签:

#### Tab 1: 横向对比 (同语言不同IPC)
- 为每种选中的语言显示性能对比图表
- 包含吞吐量柱状图、延迟柱状图、散点图
- 当数据包含校验错误时，自动展示错误数和重传次数的分组柱状图
- 直观展示同一语言下不同IPC方式的性能差异

#### Tab 2: 纵向对比 (跨语言相同IPC)
- 按IPC类型分组,对比不同语言的性能
- 箱线图展示性能分布
- 分组柱状图展示不同并发模式的表现
- 当数据包含重传记录时，自动展示各语言的校验错误数和重传次数对比图

#### Tab 3: 综合性能排名
- TOP 10最高吞吐量排名
- TOP 10最低延迟排名
- 各语言最佳IPC方式推荐表
- 多维度性能雷达图

#### Tab 4: 详细数据表
- 可选择显示任意列（含 Error_Count、Retransmit_Count）
- 支持按任意列排序
- 可下载当前视图的CSV数据

## 📊 数据指标说明

- **吞吐量 (Throughput)**: 每秒处理的消息数量 (msg/s),数值越高表示性能越好
- **平均延迟 (Avg Latency)**: 消息传输的平均耗时 (微秒 μs),数值越低表示性能越好
- **P95/P99延迟**: 95%/99%的消息在此延迟内完成,反映尾部延迟情况
- **错误数 (Error Count)**: 校验和验证失败的消息数（含错误注入产生的损坏消息），反映数据传输中的错误量
- **重传数 (Retransmit Count)**: Socket/TCP 通信中因校验失败触发的累计重传次数（共享内存无此指标）
- **Pattern**: 生产者_消费者数量配置,如 "4_4" 表示4个生产者和4个消费者
- **Message_Size**: 每条消息的大小(字节)
- **Total_Time**: 测试总耗时(秒)

## 🎯 分析场景示例

### 场景1: 找出Java语言最快的IPC方式

1. 在侧边栏只选择"Java"
2. 切换到"横向对比"Tab
3. 观察不同IPC类型的吞吐量柱状图
4. 在"综合性能排名"Tab查看推荐的最佳IPC方式

### 场景2: 比较各语言在共享内存IPC下的表现

1. 在侧边栏选择所有语言
2. 在IPC类型中只选择"shared_memory"
3. 切换到"纵向对比"Tab
4. 查看箱线图和分组柱状图,了解各语言的差异

### 场景3: 分析高并发场景下的性能

1. 在生产者数量中选择"4"或"8"
2. 在消费者数量中选择"4"或"8"
3. 观察各种图表,找出在高并发下表现最好的语言和IPC组合

### 场景4: 导出特定场景的数据

1. 设置好筛选条件(语言、消息大小、并发数等)
2. 切换到"详细数据表"Tab
3. 选择需要显示的列
4. 点击"📥 下载当前视图数据"按钮

### 场景5: 分析数据校验与重传效率

1. 在侧边栏选择所有语言和 IPC 类型
2. 切换到"横向对比"Tab，观察每种语言下方的错误数与重传数柱状图
3. 比较共享内存（仅校验、无重传）与 Socket/TCP（ACK/NACK 重传）的错误处理差异
4. 切换到"纵向对比"Tab，查看同一 IPC 类型下各语言的校验错误对比
5. 结合吞吐量数据，评估校验机制对性能的实际影响

## 💡 使用技巧

1. **性能优化**: 如果数据量很大,可以先在侧边栏缩小筛选范围,加快图表渲染速度
2. **对比分析**: 同时选择多个语言,在"纵向对比"Tab可以更清晰地看出差异
3. **导出报告**: 利用"详细数据表"的下载功能,可以将筛选后的数据导出用于进一步分析
4. **交互式探索**: 所有Plotly图表都支持缩放、悬停查看详细信息等交互操作

## ❓ 常见问题

**Q: 启动后浏览器没有自动打开?**
A: 请手动访问终端中显示的URL(通常是 http://localhost:8501)

**Q: 提示找不到CSV文件?**
A: 请确保已经运行过各语言的测试程序,并且CSV文件在`csv/`目录下

**Q: 图表加载很慢?**
A: 尝试减少选择的语言数量或缩小筛选范围,可以减少数据量

**Q: 如何添加新的测试数据?**
A: 只需将新生成的CSV文件放入`csv/`目录,文件名格式为`ipc_performance_<语言名>.csv`,重启分析器即可
